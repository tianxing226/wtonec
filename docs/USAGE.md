# Wtonec 使用教程

## 进入语音面板

1. 确认 LSPosed 已启用 Wtonec，微信已加入作用域。
2. 完全结束并重新启动微信。
3. 首次使用时等待 DEX 匹配结束。
4. 进入私聊或群聊，将输入栏切换到语音模式。
5. 长按微信语音按钮打开 Wtonec。

面板会保留当前会话、选中音色、输入内容和页面状态。发送前确认顶部显示的会话与目标聊天一致。

## 预设音色

1. 打开“预设”页。
2. 输入需要合成的文字，最大长度为 500 字。
3. 在音色搜索框输入名称或浏览列表。
4. 选择音色；需要长期使用时可加入收藏。
5. 点击试听确认内容与声音。
6. 点击“生成”仅生成音频，或点击“生成并发送”发送到当前会话。

预设模式向接口提交 `msg`、`ys` 和 `apikey`。

## 克隆音色

1. 打开“克隆”页。
2. 添加 Fish Audio 音色 ID，例如占位值 `VOICE_ID`。
3. 填写便于识别的显示名称和可选备注。
4. 保存后从自定义列表选择该音色。
5. 输入文字，执行试听、生成或发送。
6. 不再使用的条目可从列表删除。

克隆模式向接口提交 `msg`、`kl` 和 `apikey`。删除自定义条目不会删除 Fish Audio 服务方账户中的音色。

## 本地语音包

### 导入

1. 打开“语音包”页。
2. 点击导入 MP3。
3. 在 Android 系统文件选择器中选择音频。
4. Wtonec 检查 MIME、文件大小和 MP3 可解码状态。
5. 导入成功后，列表会新增条目并保存 MP3、微信语音数据和索引。

单个导入文件上限为 20 MiB。语音发送时长上限为 60 秒，较长素材应先在外部工具中裁剪。

### 管理与发送

- 搜索或排序条目。
- 点击试听确认内容。
- 使用重命名整理显示名称。
- 点击发送，将条目发送到当前会话。
- 使用导出，将 MP3 保存到系统文件选择器指定的位置。
- 使用删除，同时更新语音文件和本地索引。

跨设备迁移优先逐条导出 MP3，再在目标设备重新导入。`local-voices.json` 保存绝对路径，直接复制整个目录可能产生旧路径。

## 设置

- 配置、更新或清除自己的 API Key。
- 查看 DEX 缓存状态。
- 清理语音生成任务缓存。
- 查看语音入口与当前会话诊断。
- 查看缓存大小和运行状态。

DEX 缓存与语音任务缓存用途不同：DEX 缓存用于适配微信版本；语音任务缓存用于保存生成过程的临时 MP3 和微信语音数据。

## 数据位置

主目录：

```text
/storage/emulated/0/Android/data/com.tencent.mm/Wtonec/
```

任务缓存：

```text
/storage/emulated/0/Android/data/com.tencent.mm/cache/Wtonec/wtonec-jobs/
```

保存语音位于 `wtonec/voices/`，索引与音色配置位于 `wtonec/metadata/`，运行日志位于 `logs/`。详细文件表和清理影响见 [STORAGE.md](STORAGE.md)。

## 安全页

安全页当前展示扩展接口和开发状态。扫描执行逻辑处于后续开发阶段，页面不生成模块清单、安全评分或风险结论。

## 使用后检查

- 发送前确认当前会话未切换。
- 重要语音在清理缓存或微信数据前先导出。
- 问题发生后保留当天 `Wtonec-yyyy-MM-dd.log`。
- 微信升级后如出现 DEX 更新，完成匹配并冷启动验证一次。
- 接口失败时先检查服务状态、余额、网络和 Key 调用记录。

完整问题定位顺序见 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)。
