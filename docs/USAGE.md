# Wtonec 使用教程

## 选择宿主

Wtonec `1.5.4` 是微信模块与 QQ 模块共用的单 APK。微信和 QQ 分别维护会话、DEX 缓存、入口位置和宿主副本；配置与 canonical 语音库通过受控 Bridge 同步。

## 打开面板

### 微信

进入微信聊天页，将输入栏切换到语音模式，长按语音按钮或点击 WT 入口。

### QQ

进入 QQ NT 私聊或群聊，点击 WT 入口或使用已适配的 QQ 语音入口。

生成或发送前检查面板顶部的宿主、账号和当前会话。生成期间切换聊天时，会话校验会阻止旧任务进入新会话。

## Fish Audio

1. 打开“预设”，选择 Fish Audio。
2. 输入文字并搜索音色。
3. 选择音色后试听、生成、保存或生成并发送。

“克隆”页可以保存多个 Fish Audio 音色 ID 和显示名称。在线生成需要自己的 Tiax API Key。

## 轻颜音色

1. 在“预设”页切换到轻颜来源。
2. 搜索并选择音色。
3. 输入文字后生成、试听、保存或发送。

## 系统 TTS

1. 切换到“系统 TTS”。
2. 输入文字，可使用文字模板和语气设置。
3. 调整语速和音调。
4. 生成、试听、保存或发送。

系统 TTS 使用 Android 当前默认语音引擎，不读取 Tiax API Key。语音引擎自身的联网行为由引擎提供方决定。

## 本地与共享语音包

- 导入受支持的本地音频。
- 搜索、排序、试听、重命名、分类、导出、删除和发送。
- 微信或 QQ 保存的新语音先进入当前宿主副本，再通过受控 PFD/Binder Bridge 同步到 canonical 语音库。
- canonical 索引按 SILK SHA-256 去重，标题相同但内容不同的条目分别保留。

## 在线语音包

当前产品来源包括 Wtonec 目录、铃声多多和爱给网：

- 支持搜索、分页、试听、保存、直接发送、下载进度、取消、重试和缓存回退。
- Wtonec 目录包含公开样例和 APK 内置 fallback。
- 外部来源的页面结构、访问策略和网络状态会变化，单个来源异常不会影响本地语音、系统 TTS 和其他来源。

## 音频处理

Wtonec 在本地完成音频解码、单声道处理、24 kHz 重采样和 Tencent SILK 编码，再调用当前宿主的语音发送链。微信与 QQ 使用各自的会话和发送适配器。

## 面板和 WT 入口

- 液态玻璃：采样当前聊天背景。
- Miuix 浅色：强调清晰可读。
- AMOLED 深色：适合深色环境。
- 微信与 QQ 的 WT 入口开关和拖动位置分别保存。
- WT 入口挂载到当前聊天 Activity 内容树，不申请系统悬浮窗权限。

## 安全页

安全页读取当前设备可见的宿主作用域、模块包、进程和本地配置证据，并在本地生成报告。结果受 Android 权限、Root 状态和 LSPosed 可见性影响。
