# 安装与首次启动

## 环境要求

- Android 9（API 28）及以上设备。
- 已安装并正常运行 LSPosed。
- 已安装兼容的微信，标准包名为 `com.tencent.mm`。
- 已从 [Xposed 官方 Releases](https://github.com/Xposed-Modules-Repo/dev.wtonec/releases) 获取 APK。
- 在线生成语音时具有可用网络和自己的 Tiax API Key。

多开、工作资料或修改包名版本的微信会使用各自包名、作用域和数据目录。后续文档以标准微信为例。

## 安装 APK

1. 打开下载的 `Wtonec-v1.1.0-vc669-standard-universal-release-hardened.apk`。
2. 按系统提示完成安装。
3. 在 Wtonec 独立主页核对版本号、构建身份和模块状态。
4. 如需校验文件，参考根 README 中公布的 SHA-256。

## 配置 LSPosed

1. 打开 LSPosed 管理器。
2. 在“模块”中找到并启用 Wtonec。
3. 打开 Wtonec 作用域，只勾选准备使用的微信实例。
4. 返回系统最近任务并划掉微信。
5. 在系统“应用信息”中结束微信进程，可进一步确认冷启动条件。
6. 重新打开微信。

模块启用或作用域调整后，已在后台运行的微信进程仍保留旧加载状态，因此需要完整重启微信进程。

## 首次 DEX 匹配

首次进入微信、微信升级、降级或收到热更新后，Wtonec 会核对宿主指纹和已缓存的匹配结果。

正常流程包括：

1. 出现 DEX 缓存更新界面。
2. 按界面提示开始匹配。
3. 等待所有匹配项完成并显示成功状态。
4. 完全结束微信进程。
5. 再次启动微信，确认缓存被回读且更新界面不再重复出现。

匹配期间保持微信在前台，避免同时切换多个微信实例或清理应用数据。

## 打开语音面板

1. 进入任意私聊或群聊。
2. 将微信输入栏切换到语音模式。
3. 找到输入栏中的微信语音按钮。
4. 长按该按钮，等待 Wtonec 面板出现。
5. 确认面板包含“预设 / 克隆 / 语音包 / 设置”。

长按入口与普通点击录音是两种操作：普通点击继续使用微信原有行为，持续长按用于打开 Wtonec。

## 配置 API Key

1. 在 Wtonec 面板切换到“设置”。
2. 填写自己的 Tiax API Key。
3. 点击保存并等待保存成功提示。
4. 返回“预设”页，输入短文本进行首次试听。

Key 为空时，在线语音生成保持停用。本地已保存语音包的可用状态由文件完整性决定。获取流程见 [API_KEY.md](API_KEY.md)。

## 首次验收清单

- Wtonec 主页显示模块已激活。
- DEX 匹配完成，冷启动可以回读缓存。
- 私聊和群聊中长按语音按钮均可打开面板。
- 四个面板页面可以切换。
- API Key 保存成功。
- 预设音色列表可以加载、搜索和选择。
- 短文本可以试听、生成并发送。
- 主目录按需创建：`/storage/emulated/0/Android/data/com.tencent.mm/Wtonec/`。

## 更新版本

安装新 APK 后，重新核对 LSPosed 模块状态并完整重启微信。若新版本要求刷新 DEX 缓存，按首次匹配流程完成一次更新。升级前先导出重要 MP3，并记录当前 APK 版本和 SHA-256。

数据备份见 [STORAGE.md](STORAGE.md)，异常定位见 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)。
