# 故障排查

## 模块未激活

1. 确认 LSPosed 中已启用 Wtonec。
2. 确认作用域包含目标宿主：微信 `com.tencent.mm` 或 QQ `com.tencent.mobileqq`。
3. 完全结束目标宿主进程后重新启动。
4. 在 Wtonec 主页核对宿主版本和激活状态。

## 面板入口没有出现

### 微信

- 进入聊天页并切换到语音模式。
- 检查微信 WT 入口开关。
- 长按微信语音按钮，并查看微信 DEX 必需项目。

### QQ

- 使用 QQ NT 标准版本进入私聊或普通群聊。
- 检查 QQ WT 入口开关和 AIO 会话状态。
- QQ 更新后重新执行 QQ DEX 匹配。
- 同时启用其他 QQ 模块时，分别测试启用与停用状态并保留两边日志。

## DEX 页面重复出现

- 让匹配流程完整结束并保持宿主在前台。
- 匹配成功后完整结束并重启当前宿主。
- 微信和 QQ 使用独立缓存，排查时确认日志中的 host/package/versionCode。
- 宿主版本、split APK 或 matcher schema 变化后重新匹配属于预期行为。

## QQ 语音没有发出

- 确认当前是好友私聊或普通群聊，并核对面板顶部会话。
- 等待发送回调，避免连续重复点击。
- 检查 QQ PTT 暂存、`createPttElement`、`sendMsg`、callback 或 timeout 阶段错误。
- 保留发送失败后的 `wtonec-*.silk` 文件，供格式和波形检查。
- 记录 QQ 版本、Android 版本、LSPosed 日志以及本机气泡和对端接收结果。

QQ 内部接口随版本变化，静态构建通过不代表每个 QQ 版本都已完成真机收发验证。

## 在线音色或语音包失败

- Fish Audio/轻颜先确认 Key、余额、接口状态、文字长度和音色参数。
- 在线目录检查来源状态、网络出口和缓存。
- 单个来源失败时切换其他来源、本地语音包或系统 TTS。
- 分享日志前遮盖 Key 和签名下载地址。

## 系统 TTS 失败

- 打开系统语音设置，确认已安装语音引擎和中文语言数据。
- 首次初始化完成后再试。
- 切换引擎后重新打开面板。
- 系统 TTS 不读取 Tiax API Key。

## 导入、共享或保存失败

- 确认音频可以播放且文件内容与格式一致。
- 使用 Android 系统文件选择器重新选择。
- 检查存储空间和 Bridge 状态。
- 微信与 QQ 同步异常时，先保留两个宿主副本，再重新打开 Wtonec 独立应用触发 canonical 同步。

## 液态玻璃看不清

在设置中切换为 Miuix 浅色或 AMOLED 深色。自定义聊天背景下使用实时采样；低纹理默认背景会使用稳定的中性材质回退。

## 提交问题时附带

- Wtonec 版本、versionCode、APK SHA-256。
- Android、微信或 QQ、LSPosed 版本。
- 目标宿主、会话类型、操作步骤和准确时间。
- Wtonec 日志和对应时间段 LSPosed 日志。
- QQ 问题补充本机气泡是否出现、是否可播放、对端是否收到。

请遮盖 API Key、账号、联系人、会话内容和本地隐私路径。
